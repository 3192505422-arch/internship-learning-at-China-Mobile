## 一、原理

Playwright 是微软开发的 **Web 应用自动化测试框架**，面向自动化测试。其运行架构如下：

```
自动化程序（Playwright 客户端库）
    → 浏览器驱动（由 Playwright 项目提供）
    → 浏览器
```

---

## 二、安装

```bash
pip install playwright
```

安装浏览器（包含 Chrome、Firefox、WebKit）：

```bash
playwright install
```

也可以只安装 Chromium：

```bash
playwright install chromium
```

> 📁 浏览器默认安装在 `C:\Users\<用户名>\AppData\Local\ms-playwright`

---

## 三、简单示例

```python
# 导入同步 API
from playwright.sync_api import sync_playwright

input('1....')
# 启动 playwright driver 进程
p = sync_playwright().start()

input('2....')
# 启动浏览器，返回 Browser 类型对象
browser = p.chromium.launch(headless=False)  # headless=False 可以看到浏览器界面

# 创建新页面，返回 Page 类型对象
page = browser.new_page()
page.goto("https://www.byhy.net/cdn2/files/selenium/stock1.html")
print(page.title())  # 打印网页标题栏
```

> 💡 **提示：** `headless=False` 表示可以看到浏览器界面，方便调试。`browser` 是控制浏览器的核心对象。

### 指定浏览器路径

```python
# 通过 executable_path 参数指定使用安装的其它浏览器
browser = p.chromium.launch(
    headless=False,
    executable_path=r'c:\Program Files\Google\Chrome\Application\chrome.exe'
)
```

### 定位与操作（完整示例）

```python
# 输入通讯，点击查询 —— 这是自动化重点
page.locator('#kw').fill('通讯')   # 输入通讯
page.locator('#go').click()        # 点击查询

# 遍历并打印所有搜索结果
lcs = page.locator(".result-item").all()
for lc in lcs:
    print(lc.inner_text())

# 关闭浏览器
browser.close()

# 关闭 playwright driver 进程
p.stop()
```

## 四、自动化代码助手

Playwright 内置了 **codegen** 代码生成器，可以帮你自动产生自动化代码：

```bash
playwright codegen
```

> 🤖 根据 DeepSeek 老师的指导，也可以在 CMD 中使用以下命令启动：
> ```bash
> python -m playwright codegen
> ```

> ⚠️ **注意：** codegen 只能作为辅助工具，它主要记录人对页面的操作（输入、点击等），**并不能取代你自己编写获取页面信息的逻辑代码。**

## 五、跟踪功能 (Tracing)

启用跟踪后，Playwright 会记录自动化过程中的每个细节，生成跟踪文件用于回看调试。

```python
from playwright.sync_api import sync_playwright

p = sync_playwright().start()
browser = p.chromium.launch(headless=False)

# 创建 BrowserContext 对象
context = browser.new_context()
# 启动跟踪功能
context.tracing.start(snapshots=True, sources=True, screenshots=True)

page = context.new_page()
page.goto("https://www.byhy.net/cdn2/files/selenium/stock1.html")

# 搜索名称中包含"通讯"的股票
page.locator('#kw').fill('通讯')
page.locator('#go').click()
page.wait_for_timeout(1000)  # 等待 1 秒

lcs = page.locator(".result-item").all()
for lc in lcs:
    print(lc.inner_text())

# 搜索名称中包含"软件"的股票
page.locator('#kw').fill('软件')
page.locator('#go').click()
page.wait_for_timeout(1000)

lcs = page.locator(".result-item").all()
for lc in lcs:
    print(lc.inner_text())

# 结束跟踪，保存到 trace.zip
context.tracing.stop(path="trace.zip")

browser.close()
p.stop()
```

### 查看跟踪文件

执行完成后，当前目录下会生成 `trace.zip` 文件。有两种方式查看：

1. **在线查看：** 访问 [trace.playwright.dev](https://trace.playwright.dev) 上传 `trace.zip`
2. **本地查看：** 运行命令 `playwright show-trace trace.zip`

> 🎯 **小贴士：** 可以结合 `snapshots`（DOM 快照）、`sources`（源码）、`screenshots`（截图）三个参数，全方位记录自动化过程。
